Design Directions & Developer Handoff

This file contains component-level guidance, spacing, typography and interaction notes for implementation.

Spacing scale

Use a 4px base spacing system. Common tokens:

xs:4px, sm:8px, md:16px, lg:24px, xl:32px

Components

Buttons

  • Primary: filled with --accent-500, white text, 12px vertical padding, 16px horizontal.
  • Secondary: outline, text color --accent-700.
  • Disabled: reduce opacity to 50% and disable pointer events.

Chips (labels)

  • Small rounded rectangles using --chip-bg and --chip-text.
  • Clickable chips should have focus and hover states: outline 2px --accent-300 on focus.

Forms & Inputs

  • Inputs: 40px height, 12px padding, border-radius 8px.
  • Use labels above inputs; inline errors beneath inputs with --danger color.

States & Transitions

  • Use subtle 150ms ease transitions for hover and focus states.
  • Use snackbars for transient feedback (undo for Accept/Reject).

Responsive breakpoints

  • mobile: up to 640px — single-column, full-screen modals.
  • tablet: 641px–1024px — collapsible left rail, modal details.
  • desktop: 1025px+ — left rail persistent, detail panel slide-over.

Assets & Icons

Prefer SVG icons (16px/20px/24px). Provide an icon set (feather/heroicons) and keep icons semantically meaningful (priority icons, calendar, settings).

Developer Handoff Checklist

  1. Provide CSS tokens for colors and spacing (see `ux-color-themes.html`).
  2. Export SVG icons and provide usage examples with code snippets for each component.
  3. Include example markup for Task List Item and Task Detail in the storybook or component library.
  4. List accessibility requirements and keyboard interactions in the component docs.
